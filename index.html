Engineering Blueprint

Frontend Engineering Principles

The foundational concepts that guide every line of code I write.

01

DRY Principle

Don't Repeat Yourself. Every piece of knowledge must have a single, unambiguous, authoritative representation within the system.

  • Reusable CSS classes & custom properties
  • Modular, composable styling
  • BEM naming methodology
.card { … } /* reused everywhere */ .card__title { … } /* BEM element */ .button--primary { … } /* modifier */
02

Semantic DOM

Use the right element for the right job. Semantic HTML communicates meaning to browsers, search engines, and assistive technologies.

  • <header> — page or section header
  • <main> — primary document content
  • <section> — thematic grouping
  • <article> — self-contained content
  • <footer> — closing information
03

Separation of Concerns

Each technology layer has a single, clear responsibility — keeping codebases maintainable and scalable.

  • HTML = Structure & semantics
  • CSS = Presentation & layout
  • No inline styles. Ever.
<!-- HTML: structure only --> <button class="button--primary"> /* CSS: presentation only */
04

Grid vs Flexbox

Right tool, right job. CSS Grid and Flexbox serve different layout dimensions — using both correctly is a mark of engineering maturity.

  • Grid → macro page layout (2D)
  • Flexbox → component alignment (1D)
  • Never substitute one for the other blindly
05

Information Architecture

Structure content for humans first. A clear sitemap, logical heading hierarchy, and predictable navigation build user trust.

  • Only one <h1> per page
  • Heading hierarchy: h1 → h2 → h3
  • Anchor navigation for all major sections
06

Accessibility (A11Y)

The web is for everyone. Accessibility is not a feature — it is a fundamental engineering requirement.

  • Alt text on all meaningful images
  • Min 4.5:1 colour contrast ratio
  • Keyboard-navigable interface
  • Semantic landmark regions
